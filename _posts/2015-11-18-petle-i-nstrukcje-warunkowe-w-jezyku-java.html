Operatory logiczne</h1>
Zanim przejdę do opisania instrukcji warunkowych przeczytasz coś więcej na temat operatorów logicznych.

< jest operatorem logicznym, który znasz z poprzednich artykułów. W Javie istnieje kilka operatorów logicznych:

  • ==</code> równe,</li>
  • <</code> mniejsze,</li>
  • <=</code> mniejsze bądź równe,</li>
  • ></code> większe,</li>
  • >=</code> większe bądź równe</li>
    </ul>
    Ich działanie możesz zobaczyć na kilku przykładach poniżej:

    int x = 1;
    
int y = 1;
    
int z = 2;
    

    x == y; // true
    
x == z; // false
    
x < y; // false
    
x < z; // true
    
x <= y; // true
    
// itd.</pre>
    
Proste operacje logiczne możesz ze sobą łączyć przy pomocy dodatkowych operatorów:
    

      

    • &&</code> logiczne i,</li>

    • ||</code> logiczne lub.</li>
      
</ul>

Przekładając kilka powyższych linii na przykłady ze zmiennymi w Java wygląda to następująco:
      

      int x = 1;
      
int y = 1;
      
int z = 2;
      
x == y && z > y; // true && true => true
      
x <= y && z <= x; // true && false => false
      
x == y || z > y; // true || true => true
      
x <= y || z <= x; // true || false => true</pre>
      
Operatory w języku Java mają swój priorytet. Dzięki temu mnożenie jest wykonywane przed dodawaniem. Podobnie jest tutaj, operatory <</code>, ==</code>, <</code> itp. mają wyższy priorytet niż &&</code> czy ||</code>.
      

      Instrukcje warunkowe</h1>
      

      Instrukcja if</code>, else if</code>, else</code></h2>
      
Do tej pory każda linijka kodu, którą napisałeś w działającym programie została wykonana. W większych programach często musimy decydować co powinno być wykonane. Np. jeśli jest temperatura jest większa niż 37°C [1. Tutaj lepszym pomysłem byłyby liczby „z przecinkiem”, te omówimy w osobnym artykule.] oznacza to, że możesz być chory. Jeśli jest mniejsza niż 36°C możesz być osłabiony.
      

      To jest najprostsza instrukcja warunkowa. Blok kodu zostanie wykonany jeśli wartość w nawiasie ()</code> będzie prawdą. W przykładzie sprawdzamy, czy temperatura jest mniejsza niż 36 stopni.
      

      if (temperature < 36) {
      
    System.out.println("Jesteś osłabiony?");
      
}</pre>
      
else</code>, podobnie jak if</code>, jest słowem kluczowym. Jeśli warunek przy instrukcji if</code> nie zostanie spełniony wywołany zostanie zestaw instrukcji w bloku else</code>.
      

      if (temperature < 36) {
      
    System.out.println("Jesteś osłabiony?");
      
}
      
else {
      
    System.out.println("Masz 36 lub więcej stopni.");
      
}</pre>
      
W języku Java możesz także połączyć else</code> z if</code> jak w przykładzie poniżej:
      

      int temperature = 38;
      
if (temperature < 36) {
      
    System.out.println("Jesteś osłabiony?");
      
}
      
else if (temperature < 37) {
      
    System.out.println("Wszystko w normie!");
      
}
      
else if (temperature < 38) {
      
    System.out.println("Jesteś przeziębiony?");
      
}
      
else {
      
    System.out.println("Masz co najmniej 38 stopni! Biegiem do lekarza!");
      
}</pre>
      
Kod ten na początku sprawdza czy temperatura jest mniejsza niż 36°C. Jeśli ten warunek nie jest spełniony sprawdzamy czy temperatura jest mniejsza niż 37°C. Podobnie działa kolejny warunek. Jeśli żaden z nich nie jest spełniony zostaje wykonany kod w bloku po else</code>.
      

      Instrukcja switch</code></h2>
      
Jeśli mamy kilka warunków if</code> następujących po sobie możemy zamienić je na inną konstrukcję - switch</code>. Przejdźmy od razu do przykładu:
      

      int temperature = 37;
      
switch (temperature) {
      
    case 35:
      
        System.out.println("Jesteś osłabiony?");
      
        break;
      
    case 36:
      
        System.out.println("Wszystko w normie!");
      
        break;
      
    case 37:
      
        System.out.println("Jesteś przeziębiony?");
      
        break;
      
    case 38:
      
        System.out.println("Chyba jesteś chory.");
      
        break;
      
    default:
      
        System.out.println("Nie wiem jak się czujesz :(");
      
        break;
      
}</pre>
      
switch</code> w przykładzie powyżej sprawdza wartość zmiennej temperature</code> i w zależności od jej wartości wykonuje odpowiedni kod zdefiniowany w case</code> poniżej. W naszym przypadku na ekranie zostanie wypisane Jesteś przeziębiony?</code>.
      

      Proszę zwróć uwagę na nowe słowo kluczowe break</code>. switch</code> dopasowuje case</code> do wartość zmiennej, i wykonuje kod od pierwszego pasującego warunku do słowa kluczowego break</code>.
      

      int temperature = 35;
      
switch (temperature) {
      
    case 35:
      
        System.out.println("Jesteś osłabiony?");
      
    case 36:
      
        System.out.println("Wszystko w normie!");
      
    case 37:
      
        System.out.println("Jesteś przeziębiony?");
      
        break;
      
    case 38:
      
        System.out.println("Chyba jesteś chory.");
      
        break;
      
}</pre>
      
W przykładzie powyżej break</code> zostało pominięte przy case 35</code> oraz case 36</code>. Wartość zmiennej pasuje do case 35</code>. Przez pominięcie słowa kluczowego break</code> na ekranie zostaną wypisane 3 linijki. Dzieje się tak dlatego, że słowo kluczowe break</code> zostało wypisane dopiero przy case 37.
      

      Jesteś osłabiony?
      
Wszystko w normie!
      
Jesteś przeziębiony?</pre>
      
Warunek default</code> zostaje wykonany jeśli żadna gałąź case</code> nie pasuje do wartości zmiennej. Zachęcam Cię do eksperymentowania :).
      

      Instrukcja switch</code> jest trochę ograniczona. Sprawdzana zmienna może być zmienną kilku typów np. int</code> czy String</code>.
      

      Pętle w języku Java</h1>
      

      Pętla for</code></h2>
      
Pętla pozwala na wykonanie fragmentu kodu wielokrotnie. Na przykład jeśli chcemy wypisać wszystkie liczby od 0 do 10 możemy to zrobić przy pomocy pętli
      

      for (int number = 0; number <= 10; number++) {
      
    System.out.println(number);
      
}</pre>
      
Pierwsza linijka to nic innego jak właśnie definicja pętli for</code>. Kod wewnątrz nawiasu ()</code> możemy podzielić na trzy części:
      

        

      1. inicjalizacja zmiennych,</li>

      2. sprawdzenie warunku,</li>

      3. zmiana wartości zmiennych.</li>
        
</ol>
        
W części inicjalizującej zmienne robimy coś co już znasz, tworzymy zmienną number</code> i przypisujemy jej wartość 0</code>. Kolejna część to sprawdzenie czy wartość zmiennej jest mniejsza bądź równa 10</code>. Nowa dla Ciebie jest trzecia część a właściwie operator ++</code>. Proszę spójrz na przykład poniżej:
        

        int counter = 0;
        
counter = counter + 1;
        
counter += 1;
        
counter++;</pre>
        
Każda z trzech ostatnich linii zwiększa o jeden wartość zmiennej counter</code>. Ostatnia wersja jest najkrótsza dlatego jest najczęściej spotykana w tego typu pętlach.
        

        Kolejnym przykładem użycia pętli może być wyświetlenie każdego elementu tablicy
        

        int[] primeNumbers = new int[] {2, 3, 5, 7};
        
for (int index = 0; index < primeNumbers.length; index++) {
        
    System.out.println(primeNumbers[index]);
        
}</pre>
        
W pierwszej linijce definiujemy tablicę z kilkoma liczbami pierwszymi (jeśli nie wiesz czym są liczby pierwsze możesz przeczytać o nich w artykule na wikipedii</a>). Następnie w pętli wyświetlamy każdy element tej tablicy.
        

        Istnieje też uproszczona wersja pętli for</code>. Jeśli nie musisz mieć dostępu do zmiennej reprezentującej np. indeks tablicy możesz użyć poniżej wersji pętli.
        

        for (int primeNumber : primeNumbers) {
        
    System.out.println(primeNumber);
        
}</pre>
        
Tutaj w pierwszej linijce do zmiennej primeNumber</code> przypisujemy kolejne elementy tablicy primenNumbers</code>, które następnie wyświetlamy na ekranie.
        

        Live templates</h3>
        
Obie wersje pętli for możesz utworzyć dużo łatwiej. Z jednego z poprzednich artykułów wiesz już o istnieniu live templates. Okazuje się, że IDE pomaga nam także w tworzeniu pętli. Wystarczy, że wpiszesz fori</code> i naciśniesz Enter</code>, IDE wstawi szablon pętli za Ciebie. Jeśli w ten sam sposób chcesz wstawić szablon uproszczonej pętli for możesz wpisać I</code> i nacisnąć Enter</code>.
        

        Pętla while</h2>
        
W języku Java istnieją także inne konstrukcje pętli. Kolejnym przykładem jest pętla while</code>. Tym razem przejdziemy od razu do przykładu
        

        int number = 0;
        
while (number < 10) {
        
    System.out.println(number);
        
    number++;
        
}</pre>
        
Pętla while</code> wykonuje swoje ciało (kod wewnątrz {}</code>) tak długo jak spełniony jest warunek zapisany pomiędzy ()</code> (lub pętla nie zostanie przez nas przerwana). Podobnie jak w poprzenich przykładach pętla wyświetla zestaw liczb, tym razem liczby od 0 do 9.
        

        Poniżej kolejny przykład pętli while</code>. Tym razem warunek zakończenia pętli sprawdzany jest w jej ciele. Użyliśmy do tego celu instrukcji warunkowej if</code>. Słowo kluczowe break</code> napotkane wewnątrz pętli natychmiast przerywa jej wykonanie (dotyczy to także pętli for</code>).
        

        int number = 0;
        
while (true) {
        
    System.out.println(number);
        
    number++;
        
    if (number == 10) {
        
        break;
        
    }
        
}</pre>
        

        Nieskończona pętla</h2>
        
Pętle mają to do siebie, że przy źle zdefiniowanych warunkach mogą wykonywać kod wewnątrz swojego ciała w nieskończoność. Jest to dość częsty błąd, szczególnie spotykany na początku przygody z programowaniem. Dlatego należy uważnie sprawdzać warunki zakończenia pętli, żeby uniknąć tego błędu.
        

        Poniżej przykład pętli wyświetlającej 1</code> w nieskończoność.
        

        while(true) {
        
    System.out.println(1);
        
}</pre>
